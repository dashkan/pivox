# CLAUDE.md — web/app

Next.js 16 application with Mantine v8, Clerk.js auth, and TanStack AI.

## Tech Stack

- **Framework:** [Next.js 16+](https://nextjs.org) (App Router) — refer to https://nextjs.org/docs/llms-full.txt for complete documentation.
- **UI Framework:** [Mantine v8](https://mantine.dev) — refer to https://mantine.dev/llms-full.txt for complete API and usage instructions. @tabler/icons-react for icons, Recharts for charts.
- **Auth:** [Clerk.js](https://clerk.com) (`@clerk/nextjs`) — refer to https://clerk.com/llms.txt for usage instructions.
- **AI/Data:** [TanStack](https://tanstack.com) (AI, React Virtual) — use the TanStack MCP server tools (`tanstack_doc`, `tanstack_search_docs`) for up-to-date documentation on all TanStack libraries.
- **Language:** TypeScript 5.9, React 19
- **Package manager:** pnpm (workspace member of `web/`)

## Development CommandsUp

All commands run from this directory (`web/app/`):

```bash
pnpm dev              # Start dev server
pnpm build            # Production build
pnpm test             # Full check: typegen + prettier + lint + typecheck + vitest
pnpm typecheck        # TypeScript type checking (tsc --noEmit)
pnpm lint             # ESLint + Stylelint
pnpm vitest           # Run tests (vitest run)
pnpm vitest:watch     # Run tests in watch mode
pnpm prettier:write   # Format code
pnpm storybook        # Storybook on port 6006
pnpm analyze          # Bundle analysis build
```

## Project Structure

```

  layout.tsx          # Root layout (ClerkProvider > MantineProvider > ModalsProvider)
  page.tsx            # Home page
  protected/          # Authenticated routes (AppShell layout)
    layout.tsx        # Client component with AppShell (header, navbar, aside, footer)
    dashboard/

  api/chat/           # SSE streaming chat endpoint (TanStack AI)

ai/tools/             # AI tool definitions (weather)
proxy.ts              # Clerk middleware (Next.js 16+ uses proxy.ts, NOT middleware.ts)
env.ts                # Type-safe env vars via @t3-oss/env-nextjs
theme.ts              # Mantine theme overrides
```

## Key Patterns

- **Path alias:** `@/*` maps to project root (e.g., `import { theme } from '@/theme'`)
- **Middleware:** Next.js 16+ names it `proxy.ts` (not `middleware.ts`)
- **Env vars:** Validated with `@t3-oss/env-nextjs` in `env.ts`. Server vars (e.g., `ANTHROPIC_API_KEY`) are only accessible server-side. Client vars must be prefixed `NEXT_PUBLIC_`.
- **Clerk keys:** `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` and `CLERK_SECRET_KEY` in `.env.local`
- **React Compiler** is enabled in `next.config.ts`. Do not use `useMemo`, `useCallback`, or `memo()` unless strictly necessary — the compiler handles memoization automatically.
- **Mantine hooks:** Prefer `@mantine/hooks` over writing your own (e.g., `useMounted`, `useDisclosure`, `useDebouncedValue`). Check https://mantine.dev/hooks/ before implementing custom hooks for common patterns.
- **Next.js typed routes:** `LayoutProps<'/'>` and similar types are auto-generated by `next typegen` (included in `.next/types/`)
- **Skills:** Use `vercel-react-best-practices` when working on React components. Use `vercel-composition-patterns` when working on composite components.
